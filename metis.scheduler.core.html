start

(start m)(start config {mp-id :mp-id})

Registers a listener with the pattern __keyspace@<stmem-db>*__:<mp-id>@*@*@*. The check function becomes the listeners callback.

state

(state v)

state processes on the state vector in order to find out whats next todo. The input vector looks like this:

Example:

[{:mp-id "mpd-ref",
:struct :cont,
:no-idx 0,
:func :state,
:seq-idx 0,
:par-idx 0,
:value "ready"}]

stop

(stop {mp-id :mp-id})

De-registers the listener for the mp-id. After stopping, the system will no longer react on changes (write events) at any interface.
